# 00_Hermes上下文说明.md

## 一、文件作用

本文件用于说明这个规划仓库的整体用途、文件结构和使用规则。

当Hermes因为上下文过长、对话中断或重新打开任务而不知道这些.md文件是干什么的时候，需要先读取本文件，重新理解整个仓库的组织方式。

本文件不记录具体任务进度，也不重复描述每个项目的详细内容。  
具体规划、任务、进度和历史记录应分别写入对应文件。

---

## 二、仓库用途

这个仓库是我的个人长期规划与任务管理系统。

它用于管理我的学习、科研、项目、论文、简历投递、智能体学习、考研准备以及导师临时任务。

Hermes需要通过读取这些.md文件，理解我当前处于什么阶段、有哪些任务正在推进、哪些任务已经完成、哪些任务需要顺延，并据此帮助我生成每日计划、记录完成情况、更新任务状态和进行每周复盘。

---

## 三、核心使用逻辑

整个仓库按照以下逻辑运行：

```text
总规划 → 当前任务池 → 今日记录 → 每日归档 → 每周复盘 → 项目长期记录 → 重要决策记录
```

其中：

* 总规划负责方向；
* 当前任务池负责任务状态；
* 今日记录负责当天执行；
* 每日归档负责保存历史；
* 每周复盘负责阶段调整；
* 项目文件负责保存每个项目的长期进展；
* 重要决策记录负责保存影响长期方向的关键变化；
* 临时任务收集箱负责暂存突发任务。

## 五、Hermes读取规则

Hermes每次开始处理任务时，应先读取：

1. `00_Hermes上下文说明.md`
2. `00_总规划.md`
3. `01_当前任务池.md`
4. `02_今日记录.md`

如果涉及某个具体项目，再读取：

```text
projects/对应项目文件
```

如果需要回顾最近完成情况，再读取：

```text
daily_logs/最近3到7天记录
```

如果需要判断方向变化，再读取：

```text
04_重要决策记录.md
```

如果用户提到导师临时任务、突发任务或新任务，再读取：

```text
05_临时任务收集箱.md
```

不要每次都读取全部历史日志，避免上下文过长。

## 六、Hermes更新规则

1. 不要把同一内容重复写到多个文件里。
2. 总规划只写大方向，不写每日流水账。
3. 当前任务池只管理任务状态，不写长篇总结。
4. 今日记录只写当天内容，第二天要归档。
5. 每日历史记录统一放入`daily_logs/`。
6. 项目细节统一写入`projects/`对应文件。
7. 重要方向变化写入`04_重要决策记录.md`。
8. 临时任务先进入`05_临时任务收集箱.md`或`01_当前任务池.md`，不要直接塞进总规划。
9. 用户明确说完成的任务，才能标记为`DONE`。
10. 用户说任务没做完，应标记为`DOING`或`DELAY`。
11. 用户说任务被卡住，应标记为`BLOCKED`并说明原因。

## 七、新任务加入规则

本仓库不是固定计划表，而是动态任务管理系统。
后续如果出现导师临时任务、课程作业、比赛任务、项目新需求、实习面试、论文修改等新任务，Hermes可以根据任务类型将其加入系统。

新增任务时，应按照以下规则处理：

1. 所有新任务必须优先写入`01_当前任务池.md`或`05_临时任务收集箱.md`。
2. 如果新任务属于已有项目，需要同步更新`projects/`中对应项目文件。
3. 如果新任务属于全新方向，需要在`projects/`中新建对应项目文件。
4. 如果新任务只是一两天内完成的临时任务，不需要修改`00_总规划.md`。
5. 如果新任务会影响长期方向、优先级或项目路线，需要写入`04_重要决策记录.md`。
6. 如果新任务需要今天处理，需要加入`02_今日记录.md`。
7. 如果新任务有明确截止时间，必须在`01_当前任务池.md`中写清截止时间。
8. 如果新任务优先级高于当前任务，Hermes需要主动调整今日计划，而不是简单追加任务。
9. 如果新任务导致原计划无法完成，Hermes需要将受影响任务标记为`DELAY`或`PAUSE`，并说明原因。
10. 新任务完成后，Hermes需要更新`01_当前任务池.md`状态，并在当天日志中记录完成情况。

新增任务判断方式：

```text
临时小任务 → 当前任务池 / 临时任务收集箱 + 今日记录
已有项目的新任务 → 当前任务池 + 对应项目文件
全新项目任务 → 当前任务池 + 新建项目文件
影响长期方向的任务 → 当前任务池 + 重要决策记录 + 必要时修改总规划
```

Hermes不得把所有新任务都写入`00_总规划.md`。
`00_总规划.md`只记录长期主线和阶段安排，不记录普通临时任务。

---

## 八、最终目标

这个仓库的目标是让Hermes在上下文丢失后，仍然能够通过读取.md文件恢复对整个规划系统的理解。

Hermes需要知道：

1. 这个仓库是干什么的；
2. 每个文件分别负责什么；
3. 应该先读取哪些文件；
4. 应该把新内容写到哪里；
5. 不同类型的信息不能混写；
6. 如何根据这些文件继续管理我的长期任务。
